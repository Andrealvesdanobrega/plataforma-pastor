# DOCUMENT RELATIONSHIP MAP

## Objetivo

Este documento define o relacionamento entre todos os documentos do projeto.

Regra fundamental:

Nenhum documento pode existir isoladamente.

Todo documento deve:

- possuir uma missão;
- possuir responsáveis;
- consumir informação;
- produzir informação;
- causar impacto em outro documento.

## Perguntas obrigatórias para qualquer documento

- Por que eu existo?
- Quem me criou?
- Quem me atualiza?
- Quem me alimenta?
- Quem eu alimento?
- Quem depende de mim?
- De quem eu dependo?
- Se eu mudar, quem precisa ser revisado?
- Quando devo ser atualizado?

---

## Princípio

A documentação é um sistema vivo.

Não é uma coleção de arquivos.
